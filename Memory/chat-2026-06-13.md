## Deployment Diagnosis

The public subdomain did not open because DNS did not resolve:

```text
dsg.lorrrem.ru -> no A record
```

DNS checks showed:

```text
lorrrem.ru -> 195.133.15.12
www.lorrrem.ru -> 195.133.15.12
lorrrem.ru NS:
ns1.timeweb.ru
ns2.timeweb.ru
ns3.timeweb.org
ns4.timeweb.org
```

Conclusion:

- Files were uploaded to REG.RU hosting.
- The domain DNS was still delegated to Timeweb nameservers.
- REG.RU hosting settings did not affect public DNS until DNS delegation or DNS records were fixed.

Recommended fix discussed:

- In REG.RU domain settings, change DNS servers for `lorrrem.ru` to hosting DNS:

```text
ns1.hosting.reg.ru
ns2.hosting.reg.ru
```

Important caution:

- Changing nameservers can affect the main domain, mail, and other subdomains.
- If those are in use, DNS records should be migrated before changing nameservers.
